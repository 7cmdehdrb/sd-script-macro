min님, 드디어 이 익스텐션의 '엔진 보어업(튜닝)' 단계까지 오셨군요! 로봇 팔에 새로운 조인트(자유도)를 하나 더 추가하는 것과 똑같은 작업입니다. 

나중에 '조명(lighting)'이나 '포즈(pose)' 같은 새로운 카테고리를 뼈대에 직접 추가하고 싶으실 때, 정확히 어느 부분을 건드려야 하는지 완벽한 매뉴얼을 정리해 드립니다. 예시로 **`pose`**라는 카테고리를 새로 추가한다고 가정해 보겠습니다. 

총 3개의 구역(데이터, UI, 엔진)만 손보시면 됩니다.

### 🛠️ 1단계: 데이터 및 뷰어 업데이트 (기초 공사)
저장소와 화면에 새 카테고리가 인식되도록 리스트를 늘려줍니다. 코드 상단에 있는 3개의 함수를 수정해야 합니다.

* `load_presets()`
* `update_view()`
* `generate_cheat_sheet()`

위 세 함수 안에 있는 카테고리 리스트 `["base", "character", "cloth", "place", "etc", "main"]`를 찾아서, 맨 끝에 `"pose"`를 추가해 줍니다.
```python
# 수정 후 모습
["base", "character", "cloth", "place", "etc", "main", "pose"]
```

### 🖥️ 2단계: UI 요소 업데이트 (컨트롤 패널)
사용자가 화면에서 누르고 볼 수 있는 부분을 수정합니다. `ui()` 함수 내부를 건드립니다.

* **라디오 버튼:** `cat_radio`의 `choices` 리스트에 `"pose"`를 추가합니다.
* **템플릿 창:** `template_pos`와 `template_neg`의 `value` (디폴트 텍스트) 어딘가에 `{pose}`를 끼워 넣습니다.
* **스케줄 설명:** `schedule_input`의 `label`과 `placeholder` 텍스트에 `pose`를 추가하여 나중에 입력할 때 순서를 헷갈리지 않게 메모해 둡니다.

### ⚙️ 3단계: 스케줄 파서 업데이트 (핵심 엔진)
입력받은 텍스트를 쪼개서 변수에 담는 로직입니다. 카테고리가 6개에서 7개로 늘어났으니, 숫자를 모두 7로 맞춰야 합니다. `run()` 함수 내부를 수정합니다.

1.  **배열 크기 방어 로직 변경:**
    ```python
    while len(frag_names) < 7:  # 기존 6에서 7로 변경
        frag_names.append("")
    ```
2.  **변수 언패킹 (7개로 확장):**
    ```python
    # 기존 코드 맨 끝에 po_n(포즈용 변수)을 추가하고 7개까지 자릅니다.
    b_n, ch_n, cl_n, p_n, e_n, m_n, po_n = frag_names[:7] 
    ```
3.  **포맷팅 주입 (템플릿에 데이터 쏘아주기):**
    `raw_pos`와 `raw_neg`를 조립하는 `.format(...)` 안에 방금 만든 `pose` 변수를 연결해 줍니다.
    ```python
    raw_pos = template_pos.format(
        base=get_text("base", b_n, True), 
        # ... (기존 코드들) ...
        main=get_text("main", m_n, True),
        pose=get_text("pose", po_n, True) # <-- 새로 추가!
    )
    ```
    *(부정 프롬프트인 `raw_neg` 쪽은 `True` 대신 `False`로 적어주시면 됩니다.)*
